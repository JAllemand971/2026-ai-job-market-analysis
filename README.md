# 2026 AI Job Market Analysis

## Overview

This project is an end-to-end data science and machine learning analysis of the 2026 AI job market.

The goal is to transform a noisy job posting dataset into a structured market intelligence dataset that can be used to analyze:

- AI and Data job categories
- Required technical skills
- Salary trends
- Country-level hiring patterns
- Remote and hybrid work distribution
- Skill gaps and market signals
- Machine learning patterns in job descriptions

The project combines data cleaning, feature engineering, job title normalization, ESCO occupation mapping, skill enrichment, local LLM validation, exploratory data analysis, data visualization, and machine learning.

This is not only a visualization project. It is a complete applied AI and data pipeline designed to convert messy job postings into structured insights.

---

## Why This Project Matters

AI job postings are difficult to analyze because they often contain:

- noisy job titles
- incomplete skill fields
- inconsistent descriptions
- missing salary information
- unclear remote status
- overlapping job categories
- modern roles that do not always map cleanly to traditional occupation taxonomies

For example, similar roles can appear under many different titles:

- AI Engineer
- Generative AI Engineer
- LLM Engineer
- Machine Learning Engineer
- Applied Scientist
- AI Software Engineer
- MLOps Engineer

At the same time, many postings do not explicitly list required skills, even when the job description strongly implies them.

This project addresses these problems by building a structured analysis pipeline that:

- cleans and standardizes raw job market data
- normalizes job titles into a custom AI job taxonomy
- maps roles to ESCO occupation labels when possible
- enriches missing skills using similar job postings
- validates inferred skills with a local LLM through Ollama
- preserves traceability between original, inferred, validated, and final recommended skills
- produces analysis-ready datasets for market insights and machine learning

## Dataset Summary

The final dataset contains:

- 5,773 AI-related job postings
- 5 countries
- 16 normalized job categories
- salary information when available
- job titles and descriptions
- company, country, city, experience level, and remote status
- original required skills when provided
- enriched recommended skills generated by the skill completion pipeline

The original dataset had limited skill coverage. Only a small portion of postings contained explicit required skills.

To make skill demand analysis more useful, the project creates a final recommended skill signal by combining original skills, similarity-based inference, local LLM validation, and rule-based rescue logic.

# Part 1: Building the Skill Dataset

## What `01_build_skill_dataset.ipynb` Does

The first notebook is the core data engineering and skill enrichment pipeline.

It turns the raw job posting dataset into a clean, structured, analysis-ready dataset.

The notebook performs the following steps:

1. Raw data exploration
2. Data cleaning
3. Non-English job title cleaning
4. Job title normalization
5. Custom AI job taxonomy creation
6. ESCO occupation mapping
7. KNN-based skill inference
8. Local LLM validation with Ollama
9. Rule-based rescue logic
10. Final recommended skill selection
11. Export of the final dataset

---

## Data Quality Problem

The original dataset contains useful job market information, but it is not directly ready for analysis.

The main problems are:

- many missing skill fields
- incomplete salary fields
- noisy job titles
- non-English job titles
- truncated job descriptions
- inconsistent remote status
- overlapping AI and data job categories

The project starts by measuring which fields are reliable enough for analysis.

![Dataset Coverage](outputs/figures/001_dataset_coverage_by_analysis_field.png)

The coverage analysis shows that basic job information is strong, but original skill coverage and salary coverage are weaker.

This is why the project does not rely only on the raw dataset. It builds an enriched dataset before performing market analysis.

# Key Findings

## 1. The AI job market is mostly applied and engineering-driven

The market is not only composed of research jobs.

Many postings are focused on applied AI, data platforms, software engineering, cloud systems, LLM applications, and production deployment.

This suggests that candidates should combine AI knowledge with practical engineering ability.

---

## 2. A few role families dominate the market

The market is concentrated around major categories such as:

- AI Engineer
- Data Scientist
- Machine Learning Engineer
- Software Engineer
- LLM Engineer
- Research Scientist
- MLOps Engineer

This means job seekers should understand broader role families rather than focusing only on exact job titles.

---

## 3. Python remains the core technical skill

Python appears as one of the strongest skill signals across the dataset.

It remains a foundational skill for AI engineering, machine learning, data science, and applied AI roles.

---

## 4. Cloud and production skills matter

The market shows demand for cloud platforms and deployment-oriented tools.

Skills such as AWS, Azure, Docker, Kubernetes, and MLOps-related tooling appear as important signals for production AI roles.

This suggests that AI professionals are expected to move beyond notebooks and understand how AI systems are deployed.

---

## 5. Generative AI is becoming part of applied engineering roles

Skills related to modern AI systems appear in the dataset, including:

- RAG
- LLMs
- OpenAI tooling
- LangChain
- transformers
- vector-style retrieval workflows

This suggests that GenAI is increasingly connected to real production jobs.

---

## 6. Skill combinations matter more than isolated skills

The skill co-occurrence analysis shows that companies usually ask for combinations of technologies.

Examples include:

- Python + NLP + RAG
- cloud + Docker + Kubernetes
- SQL + Spark + data platforms
- PyTorch + transformers
- MLOps + cloud deployment

This makes stack-level understanding more valuable than learning isolated tools.

---

## 7. Salary analysis must be interpreted carefully

Salary data is useful but incomplete.

Salary coverage varies strongly by country, and salary values can be noisy.

The project therefore separates salary analysis from general market analysis and uses medians, distributions, and sample-size awareness.

---

## 8. Mid-level and senior roles dominate

The experience-level analysis suggests that the AI job market is more favorable to candidates with professional experience.

Junior roles exist, but they represent a smaller portion of the dataset.

This makes portfolio projects, applied skills, internships, and practical engineering experience especially important for entry-level candidates.

# Project Limitations

This project is exploratory and should not be interpreted as a complete representation of the global AI job market.

Main limitations:

- The dataset may not represent all AI jobs globally.
- Some job descriptions are truncated.
- Original required skill data is incomplete.
- Some final skills are inferred, not directly provided by employers.
- Salary data is incomplete.
- Salary coverage varies strongly by country.
- Salaries should not be compared across countries without currency and cost-of-living normalization.
- Remote status is often unspecified.
- Job categories can overlap.
- Some job postings may be semantically duplicated even if exact duplicate rows were removed.
- Machine learning models are exploratory and not causal.

Despite these limitations, the project provides a useful structured view of AI job market patterns.
